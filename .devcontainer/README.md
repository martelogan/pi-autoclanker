# Devcontainer

This is an optional container-first contributor path.

Use this when:

1. You want stronger host isolation than the default local tooling.
2. You already work in VS Code Dev Containers or Codespaces-style workflows.

Quickstart:

1. Open the repository in a devcontainer-capable editor.
2. Reopen in the container.
3. Run `bin/dev doctor` and `bin/dev test`.

Notes:

1. `bin/dev` remains the command surface inside the container.
2. This mode is optional; the default local workflow is still `bin/dev`.
